BASBL, Branch-And-Sandwich BiLevel, is a MINOTAUR solver for nonlinear bilevel problems.

Solving nonlinear bilevel problems to global optimality is a long-standing challenge in optimization theory. Special cases of bilevel problems, such as problems of linear (quadratic) â€“ linear (quadratic) type, have been studied extensively and many algorithms have been proposed in the literature (Dempe, 2002; Floudas and Gounaris, 2009). However, the general nonconvex form is very challenging and only recently the first deterministic method was proposed by (Mitsos et al., 2008).

BASBL is based on a recent Branch-and-Sandwich algorithm (see References section for the theory behind him) which guarantees to solve a broad class of bilevel problems to global optimality. Moreover, BASBL is enhanced with several new techniques: reduction of overhead of the UB procedure, alternative ways to calculate tighter bounds, as well as alternative choices for branching and node selection.

Running BASBL

Software requirements

In order to use MINOTAUR/BASBL, users will need to have a MINOTAUR installed (See Minotaur_Download). MINOTAUR is completely open-source and provides interfaces for AMPL and classical branch-and-bound algorithms.

BASBL solver's prerequisites

BASBL is capable of solving nonlinear bilevel programming (NLBP) models. In order to achieve convergence to global optimality:

  • Explicit bounds must be known for all variables.
  • All the functions involved in (NLBP) must be bounded from below and/or above and continuous on X x Y.
  • A constraint qualifications holds for the inner problem for all x values.

Output Fields

Field Description
Iter The number of the current iteration
Fopt The outer objective value
fopt The inner objective value
(Xopt, Yopt) incumbent solution point
ILB Total number of solved Inner Lower Bounding subproblems
IUB Total number of solved Inner Upper Bounding subproblems
LB Total number of solved (outer) Lower Bounding subproblems
UB Total number of solved (outer) Upper Bounding subproblems
OVF Total number of solved Optimal Value Function subproblems
L Total number of open nodes in the Branch-and-Sandwich tree
Lin Total number of inner open nodes in the Branch-and-Sandwich tree
GAMS CPU time reported by GAMS solvers'
BASBL the total sum spent on GAMS and running the main BASBL program

Termination Messages

BASBL will issue one of the following statements:

*** Successful termination! ***

This is the most desirable termination status. The bilevel problem has been solved within tolerances in this case.

*** Exceeded maximum number of iterations ***

The global optimality is not guaranteed in this case. BASBL will terminate with this message when the --basbl_iter_limit criterion set by the user was reached.

*** Failed ***

BASBL failed to find feasible/global solution.

BASBL Options

The BASBL options allow the user to control termination tolerances, branching and node selection strategies, heuristic for upper bounding procedure, and output (log) options as detailed in this section.

Options Style

In MINOTAUR toolkit options (see Minotaur_Options) by default are specified as option-name and option-value pairs. e.g. --basbl_iter_limit 100 sets the option-name --basbl_iter_limit to a value of 100. The option-names are case insensitive. So --basbl_iter_limit 100, --BASBL_iter_limit 100, --basbl_ITER_LIMIT 100 are all equivalent. Option values on the other hand, are case sensitive. So --basbl_global_solver BARON, is different from --basbl_global_solver baron.

BASBL Options

Option Description Default
--basbl_outer_tol Outer optimality tolerance 1e-3
--basbl_inner_tol Inner optimality tolerance 1e-5
--basbl_global_solver Global NLP solver GAMS/BARON

--basbl_ub_heuristics Use UB heuristic true/false true
